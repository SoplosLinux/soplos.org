Package Creation

How Soplos packages its software and how to propose a new one.

Contents

  • Overview
  • Soplos Packager
  • Propose a Program

Overview

The Soplos repository is strictly curated. Every package is built, signed and maintained internally by the developer. There are no third-party uploads or community-built packages — everything in the repo goes through the same internal process.

Soplos Packager

Soplos Packager is the internal tool used to create .deb packages for the Soplos repository. It handles the full packaging workflow:

  • Debian package structure and metadata
  • AppStream metainfo generation
  • GPG signing
  • Repository integration via Soplos Repo Manager

Propose a Program

If you have developed a program and would like to share it with the Soplos community, you can propose it. We will test it and, if it is viable, package it and publish it in our repositories.

You can send your proposal through any of these channels:

  • GitHub Issues — github.com/SoplosLinux
  • Soplos Linux Forums — soplos.org/forums
  • Telegram community group
Note: Not all proposals will be accepted. Inclusion depends on relevance, quality and compatibility with the distribution.
