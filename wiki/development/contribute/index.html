Contribute

Bug Reports

Found something that does not work as expected? You can report it through either of these channels:

  • GitHub Issues: Open an issue in the corresponding repository at github.com/SoplosLinux. Include your distribution version, a description of the problem and steps to reproduce it.
  • Forums: Post in the support section of the Soplos Linux forums. The community and the developer read every thread.

Translation

Soplos Linux and its applications are available in 8 languages. If you want to contribute a new language or improve an existing translation, get in touch first — translation is a coordinated effort, not an open submission.

Contact via any of these channels before starting:

  • Soplos Linux forums
  • GitHub Issues
  • Telegram community group

Testing

Pre-release versions of Soplos Linux are tested by a small group of committed users selected by the developer. Testing access is not open to everyone — it requires a demonstrated commitment to the project.

Note: Test builds are for people who will actually test and report. If you are looking for early access to new versions, this is not the right channel.

Donations

Soplos Linux is developed and maintained by a single person in their free time, with no corporate backing. Financial contributions help cover server costs, hardware and development tools.

You can donate via the Donations page.
